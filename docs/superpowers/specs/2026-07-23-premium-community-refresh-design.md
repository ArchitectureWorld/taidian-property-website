# 泰典物业官网视觉优化设计

## 目标

在不改变纯静态技术栈和现有业务主线的前提下，完成一轮可直接上线预览的视觉重构。页面需要从“概念化建筑科技演示”转向“专业、可信、温润、长期主义”的住宅物业品牌表达，并重点改善移动端传播体验。

## 现状审查

1. 首屏同时使用超大品牌字、建筑坐标、网格、技术标注和大幅暗色照片，视觉噪音偏高，物业服务的可信感与亲和力反而被削弱。
2. 唯一的 `community.webp` 分辨率为 640×246，并且画面内已有标题和数据文字；作为全屏背景放大会暴露清晰度和模板感。
3. 案例区三张卡片重复同一张图、同一构图和同一尺寸，缺乏节奏与重点。
4. 深色章节连续出现，页面层次主要依赖字号和动效，色彩与材质变化不足。
5. 移动端直接隐藏主导航，没有可访问的菜单替代；联系入口也不够持续可见。
6. 示例数据和替换提示直接进入核心文案，影响正式品牌观感。

## 视觉方向：社区里的长期主义

- 主色使用温暖米白、深森林绿和克制铜棕，替代高冷蓝黑科技风。
- 中文排版采用编辑式大标题与清晰正文层级，强调“真实生活、可靠判断、长期服务”。
- 只在少数关键容器使用圆角；其余通过细线、留白和对齐建立秩序。
- 删除坐标、网格、工程轮廓和伪数据标注，不再用“科技感”证明专业。
- 将低分辨率社区图限制在有边界的视觉窗口内，使用裁切、叠色与局部放大降低素材缺陷。
- 动效仅用于轻量显现、导航状态和图片微位移，并完整支持 `prefers-reduced-motion`。

## 信息架构

1. **导航**：品牌、五个章节链接、联系电话；移动端提供菜单按钮和抽屉式导航。
2. **首屏**：一句核心价值主张、业务定位、两个行动入口，以及经过裁切的社区影像。
3. **服务证明带**：用三条非虚构的能力描述替代缺少依据的统计卡片。
4. **经验章节**：保留“16年+”演示位，但明确为待核验数据，并把重点放在经验如何转化为判断。
5. **价值章节**：用三条纵向服务原则解释“高性价比”，避免口号堆叠。
6. **老旧社区章节**：展示真实场景理解和可执行的服务动作。
7. **案例章节**：改为一张重点案例加两张辅助案例的非对称编排；继续保留真实资料替换位。
8. **联系章节**：以强对比行动区收尾，同时集中展示演示信息声明。

## 内容原则

- 保留“泰典物业、住宅物业经验、老旧社区、高性价比、每一分投入”五项核心信息。
- 不新增无法核验的满意度、响应时长、项目数量或住户数量。
- “16年+”继续显示，但必须附带“示例数据，正式发布前核验”的小标签。
- 正式联系电话、项目名和地址仍使用占位值，但提示集中在联系区与页脚，不打断主要叙事。

## 实现约束

- 继续使用纯静态结构：`index.html`、分层 CSS（基础、章节、响应式）和 `script.js`。
- 不引入框架、构建工具、CDN、外部字体或外部图床。
- 保留 `assets/community.webp`，不新增未经授权的第三方图片。
- 兼容桌面端、平板、手机和微信内置浏览器。
- 所有交互必须支持键盘操作和明确的 ARIA 状态。
- 页面无横向溢出，脚本无运行时错误，静态验收测试全部通过。
